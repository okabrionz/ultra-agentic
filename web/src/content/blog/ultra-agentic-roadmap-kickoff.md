Ultra Agentic is starting as a catalog first. Before building more surface area—more pages, more integrations, more polish—the priority is a small, honestly labeled set of MCP servers, skills, and datasets that describe what agent infrastructure actually looks like when it is not oversold. The website exists to make that catalog browsable and to explain the labeling policy behind it, not to announce a finished product.

That means the current entry list is intentionally mixed in maturity. Some entries—repository inspection, documentation retrieval, deployment readiness—are in beta, with a versioned ZIP and source path a reader can pull down and inspect today. Others—database context, observability triage, a tool-failure dataset—are still planned: specified in enough detail to be useful as a roadmap item, but with no artifact yet. Both states are shown side by side rather than hiding the planned entries until they are done.

This early stage does not include a hosted, production deployment of anything in the catalog. There is no live endpoint to point a client at yet. What exists is the catalog itself, the beta downloads for entries that have them, and the documentation describing intended scope for entries that do not. Anyone evaluating the project today should expect a set of inspectable artifacts and specifications, not a running service.

The near-term roadmap is straightforward: bring more planned entries to beta, keep beta entries honest about what still might change, and only add "stable" labels once a maintained release contract is actually in place. A blog will track that progression alongside a get-started guide and catalog policy notes.

If a claim in the catalog ever looks ahead of what is actually shipped, that is worth flagging—keeping the roadmap and the catalog state aligned is the point of starting this way.
